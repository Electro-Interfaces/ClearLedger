## 3. Слой 1 — Первичные документы

**Что:** Все входящие данные из всех источников, приведённые к формату первичных документов.

**Откуда приходят:**
- **Источники** → автоматически через API (STS сменные отчёты, ТТН, процессинг)
- **Файлы и документы** → ручная загрузка (PDF, Excel, сканы, Word, XML)
- **Обработки** → правила, которые превращают сырые данные в структурированные документы
- **1С** → справочники, закрытые периоды, историческая бухгалтерия

**Git-like версионирование:**
- Каждый документ имеет историю версий
- Последняя версия отображается по умолчанию
- Предыдущие версии доступны для просмотра
- Идеология контроля версий (репозиторий, ветки, коммиты) пронизывает весь проект
- Документы могут содержать ошибки — корректировка происходит в Слое 2

**Хранение:** IndexedDB (клиент) / PostgreSQL (сервер), immutable оригиналы в файловом хранилище.

## 4. Слой 2 — Нормализация и аналитика

**Что:** Мост от первичных документов к внутренней БД. Ядро системы.

**Процессы:**
1. **Нормализация форматов** — приведение к единой структуре
2. **Пересчёт единиц** — кг → литры (плотность), штуки → кг (весовой товар)
3. **Маппинг номенклатуры** — коды POS → единый справочник (АИ-92, АИ-95, ДТ)
4. **Маппинг каналов оплаты** → виртуальные склады (Яндекс, Карты, Ведомости)
5. **Упаковка для AI** — данные представлены для эффективной работы с нейросетями

**Сверка с 1С:**
- По закрытым периодам — обязательная сверка с данными из 1С
- Расхождения выявляются и документируются

**Периоды:**
- **Открытый период** — свободная корректировка документов
- **Закрытый период** — корректировка с ограничениями, разрешениями, аудитом

**Аналитика во всех разрезах:**
- Управленческий учёт
- Бухгалтерский учёт
- Налоговый учёт
- Финансовый анализ

## 5. Слой 3 — Экспорт в 1С

**Что:** Подготовленные документы для загрузки через ElsyPlus Ledger.

**Процесс:**
1. Из нормализованного слоя бухгалтер выбирает документы для выгрузки
2. Документы сверяются с 1С по закрытым периодам
3. Формируется пакет для загрузки через ElsyPlus Ledger
4. ElsyPlus Ledger создаёт документы 1С с проводками
5. В 1С — проводки, отчётность для налоговой и других контуров

**Связь с ElsyPlus Ledger:**
- Текущий загрузчик (v5.2) берёт данные напрямую из STS API
- В будущем переключается на GIG Fuel Ledger как источник подготовленных данных
- Данные проходят полный конвейер: обработка → нормализация → подготовка → загрузка

## 6. Обработки — ключевая сущность

Обработка — это набор правил/скриптов трансформации, который:
- Берёт сырые данные (из API или файлов)
- Применяет правила парсинга, валидации, маппинга
- Создаёт структурированные первичные документы в Слой 1

**Примеры обработок:**
- «Сменный отчёт STS» — парсит JSON из API, создаёт документ смены
- «ТТН из Excel» — парсит Excel-файл, создаёт ТТН
- «Банковская выписка» — парсит CSV/XML из банка
- «Акт сверки» — парсит PDF/Excel

Обработки могут быть:
- **Предустановленные** — идут с системой
- **Пользовательские** — настраиваются под конкретные форматы

## 7. Принципы

1. **Git-like версионирование** — контроль версий на всех слоях
2. **Периоды** (открытый/закрытый) — ключевой принцип работы с данными
3. **Бухгалтер = контроллер** — финальное решение за бухгалтером
4. **Универсальность** — не только топливо, бухгалтер работает в едином поле
5. **Сверка с 1С** — обязательная по закрытым периодам
6. **AI-ready** — данные упакованы для эффективной работы с нейросетями
